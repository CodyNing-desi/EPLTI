# EPLTI 升级优化方案 (Plan 2.1) —— 剑指 Xiaomi MiMo Orbit 百万亿 Token 计划

基于 `MIMOPLAN.md` 的架构，本方案 (Plan 2.1) 旨在进一步优化系统性能、提升用户体验，并强化项目在“Xiaomi MiMo Orbit 百万亿 Token 创造者激励计划”中的核心竞争力。

## 一、 核心升级目标

在原有“静态评分 + 动态 AI 解读”的基础上，增加**全局数据洞察**能力，优化 AI **响应体验与成本控制**，确立项目作为**“动态体育情绪大模型应用标杆”**的定位。

---

## 二、 产品与增长层面优化 (Product & Growth)

### 2.1 引入“全网赛季情绪指数” (Elevate P2 to Core Feature)
传统的心理测试是“孤岛式”的个人体验。为了体现产品的高活跃度和数据价值，我们需要引入群体数据洞察。

*   **功能描述**：在首页或独立面板展示全网球迷的情绪大盘。例如：
    *   *“本周阿森纳球迷平均 R（韧性）指数下跌 40%，达到赛季冰点。”*
    *   *“双红会后，曼联球迷的 E（激情）指数超越全网 90% 球迷。”*
*   **技术实现**：后端在记录测试结果时，额外聚合并统计各主队球迷的五维均值变化。每周生成一份简报。
*   **战略意义**：这不仅极大地增强了项目的社交分享价值（从“看自己”变成“看热闹”），还能在 MiMo Orbit 申请中证明该项目具备**海量数据处理与社会化情绪分析**的潜力。

### 2.2 AI 社交裂变点强化 (Shareable Moments)
*   **专属吐槽海报**：目前的动态海报文案（AI Poster Line）非常好。在此基础上，允许用户将 AI 生成的“扎心金句”一键生成带有 Xiaomi MiMo Logo 水印的战报图，分享至朋友圈/虎扑。这不仅促进裂变，也能作为使用 MiMo 模型的有力证明（Logo 露出）。

## 三、 技术与架构优化 (Technical & Architecture)

### 3.1 引入 Redis 多级缓存策略 (Cost & Concurrency Control)
高并发下全量请求 AI API 是极其昂贵且高风险的。

*   **痛点**：如果 1000 个“曼联球迷+INTJ型”用户在同一轮次测试，不应调用 1000 次 API 生成相同的背景解读。
*   **解决方案**：
    *   **缓存 Key 设计**：`insight:{gameweek}:{type_code}:{detected_team}`
    *   **逻辑**：当用户请求 AI 解读时，先查 Redis 缓存。若命中，直接返回（响应时间 <50ms）。若未命中，再调用 MiMo API 生成，并写入缓存（设置过期时间为当前赛事周结束）。
    *   **对话 Agent 缓存**：聊天记录 (Session History) 必须使用 Redis 管理，避免内存泄漏，同时保证用户刷新页面后对话不丢失。

### 3.2 全面升级流式输出体验 (Streaming UX)
*   **痛点**：等待大模型返回 800 tokens 可能需要 3-5 秒，用户容易流失。
*   **解决方案**：
    *   无论是“个性化解读”还是“Agent 陪聊”，全部由普通的 HTTP POST 改为 **Server-Sent Events (SSE)** 流式传输。
    *   前端实现“打字机”效果，提升 AI 交互的真实感和沉浸感。

### 3.3 Prompt 工程进阶：Few-Shot (少样本提示)
*   **痛点**：基础 Prompt 可能导致大模型生成过于“书面化”或“官方”的回答，缺乏球迷社区特有的“梗”文化。
*   **解决方案**：在系统提示词 (System Prompt) 中加入典型的“虎扑/懂球帝老哥”语录作为范例。
    *   *示例注入*：“请模仿以下风格：‘就这防线，我奶奶上去都能防得住。’、‘别问，问就是争四狂魔DNA动了。’”

## 四、 针对 MiMo Orbit 计划的申请策略与话术 (Pitching Strategy)

在填写 Xiaomi MiMo Orbit 申请表时，建议采用以下核心叙事逻辑：

1.  **首创“动态事件驱动” (Event-Driven Prompting) 的 AI 应用范式**
    *   *话术*：“与市面上 99% 的静态套壳 AI 不同，EPLTI 的底层引擎是真实世界的体育赛事数据。我们的 Prompt 会随着现实比赛的胜负实时变动，让 AI 真正做到‘感知世界，与时俱进’。”
2.  **极高的用户活跃度与明确的 Token 消耗模型**
    *   *话术*：“依托庞大的英超球迷基数和高度情绪化的赛事周期，项目预估日均 API 调用量级在 100W+ Tokens，德比战/争冠战等关键节点峰值可达 500W Tokens。这是一个天然的高粘性、高消耗模型。”
3.  **助力 MiMo 展现“情感分析与拟人化交互”的硬实力**
    *   *话术*：“体育赛场是人类极端情绪的放大器。EPLTI 完美验证了 Xiaomi MiMo 模型在处理复杂球迷情绪、输出高质量幽默文案以及进行深度角色扮演（Agent）方面的卓越表现。”
